ChatGPT-通用人工智能?

Introduction

今年一月份，openai 公司的人工智能应用 ChatGPT 的用户量突破 $10^8$，引起公众对人工智能技术的热烈讨论。而ChatGPT 是一个 LLG(Large Language model)，通过读取上文来猜测生成相关的文本。

在不少人看来，ChatGPT 已经是一个相对完善的人工智能，ChatGPT 的改进版本 GPT4.0，有望实现真正意义上的通用型人工智能。

本文将简单探讨 ChatGPT 的原理并做一些测试，对一些未来的可能做出推测。

Principle Analysis

ChatGPT 的全称是 Chat General Prompt Transformer，中文翻译为 “对话通用引导转换器”。它使用了基于神经网络算法的深度学习技术。

General Pinciple

ChatGPT 的基本工作方式是通过给定的上文（Prompt），根据自身训练的数据，来推测出一个合理的下文。自然的，用它续写文章是非常合理的事情。

下面我们举一个用它聊天的例子。

这是一段幻影彭一个人工智能的聊天记录：

幻影彭：你是人工智能吗?

AI：是的

除了 “是的” 之外，其它都是作为 Prompt 输入给上文猜下文的程序，然后 “AI:” 后面生成 “是的” 这个肯定回答是很合理的。而如果将斜体部分设置为用户不可见，那么给用户的感受就是一个聊天机器人。

总而言之，通过合理的场景设计，ChatGPT 能用一个基本的 “给上文猜下文” 的功能，实现我们想要的几乎所有功能。

Neural Network

上面提到了 ChatGPT 所有的人工智能功能都是通过”给定上文预测下文”的方式实现的，那么神经网络算法又是如何实现 “预测下文” 的呢？

神经网络算法大约于 1940 年被提出，是模仿人类大脑得到的一项成果。单层的神经网络可以完成线性分类任务，多层神经网络通过隐藏层对输入向量的坐标空间进行变换，能够拟合任意连续函数。这里的函数定义域不局限于 一维的 $x$ 坐标，而可以是任意维的向量，值域同理。

这是单个神经元的结构，其中 $w_1,w_2,w_3$ 是”树突”部分，$\sum$ 是”细胞核”，$b$ 可以视为”细胞核”的一部分，$f(x)$ 是”轴突”部分。神经元根据树突部分的输入，计算 $f(a_1w_1+a_2w_2+a_3w_3+b)$ 并输出。$f$ 函数一般取 Sigmoid 函数 $f(x)=\frac{1}{1+e^{-x}}$（这个函数就是高中生物里那个 “S” 型曲线的图像），或者 ReLU 函数 $f(x)=\max(0,x)$。前者是早期使用的一个非线性函数，后者(ReLU函数)是研究生物大脑神经元激励机制后提出的一个函数，大部分场景的效果要好于前者，这也体现了神经网络算法和生物大脑神经元结构的强相关性。

通过设置适当的 $w,b$ 参数，以及适当的神经元结构，这个系统能够很精确的预测一个函数的结果。那么通过预测函数结果怎么实现人工智能呢？

举个简单的例子，我们希望有一个人工智能能够识别一个手写的数字，那么先把数字转化为 $24*24$ 大小的黑白图像。然后设计这样一个函数，它的参数一共有 $576$ 个，每一个参数是 $0$ 或 $1$，代表这个像素点是黑还是白，函数值是这个数字。如果神经网络能够精确的预测出这个函数的结果，那么它自然就实现了识别手写数字的功能。

一个真正能够应用的神经网络可能会有上万的这样的神经元，每一层的神经元之间都是两两有连边，需要设计的 $w,b$ 参数太多了，不可能通过人类完成。神经网络算法的精髓就是通过数据的输入让程序自己去设定 $w,b$ 参数，这个过程就是所谓的”训练”。

Intelligence?

在笔者看来，ChatGPT 不是真正意义上的通用人工智能，根据上面神经网络工作的原理，很容易设计出 ChatGPT 无法正确处理的问题。

Is it prime?

神经网络的本质是通过复杂的线性坐标空间变换来拟合函数，我们只需要设计一个不容易拟合的函数就可以让神经网络罢工，比如判断一个数是否是质数。

Plus And Multiplication

ChatGPT 能很好的回答加法问题，但是无法处理乘法问题。

笔者各自随机生成了 $5$ 个结果在 $[10^4,4\times 10^4]$ 以内的加法和乘法运算问题。ChatGPT 正确回答了全部的加法问题但错误回答了所有的乘法问题，错误的方式是再某一位错误，例如 143 * 116 = 16588，ChatGPT 的回答是 16568。

我们从数学角度简单分析一下原因。以下是加法和乘法的函数图像：

对于神经网络来说，越线性的图像，越容易被拟合，二元加法或者说是三元一次方程组 a+b=c 是完全的线性函数。而二元乘法通过线性拟合很难做到精确。

至于上例中是否是质数的函数图像，它几乎不可能被神经网络拟合——它根本无法求导。

A Child

我们在讨论一个事物的”智能”时，到底什么才是我们的评判标准？我们真的关心它的本质是什么吗？

实际上，我们只关心它是否能对我们的交互产生合理的回应。这也是 ChatGPT 如此 amazing 的原因。在很多问题上，ChatGPT 确实能做出很正确的回应，就像一个真正的人那样。

假设 openai 只是雇佣了一大批员工来对我们的问题做出正确回应，在我们不知道这一点的情况下，仍然会对 openai 的先进”人工”智能技术感到不可思议。

同样的，假设身边的某个同学的大脑实际上是一个机器，在不能切开他的脑袋看看的前提下，我们仍然认为这位同学是一个真正的”人”。

笔者看来，通过对网络中海量数据的分类和学习，ChatGPT 表现出的逻辑能力可能与 6 到 7 岁的儿童相当。

openai 宣布 ChatGPT 的下一代 GPT4.0 将会使用高达 $10^{14}$ 条数据进行训练。$600$ 倍于 ChatGPT 的训练数据能带来多大提升呢？笔者持悲观的态度——不会有本质性的飞跃。神经网络框架的人工智能数据量需求关于智能程度的函数增长很可能是超越指数函数的。

What’s the future?

基于神经网络的人工智能确实存在着无法逾越的限制，笔者不看好神经网络在通用人工智能上的表现，但它们仍然可以在特定领域发挥重要作用，并与其他方法相结合，创造更先进、更强大的人工智能系统。

举例来说，在判断质数的问题上，ChatGPT 明显能认识到问题是什么，只要有人为它提供一个调用外部代码的功能，它就可以正确的做出回应。

因此这种人工智能很大程度上无法做到脱离人类。必须要有人为它写成一个判断质数的代码，它才有可能去处理自然语言的询问并调用对应代码。

距离 ChatGPT 发布还只过去了 3 个多月，它的设计者可能还没有准备好 “判断质数的函数”，当这个 “判断质数的函数” 被写好时，ChatGPT 可能可以通过外部调用这个函数来完成 “判断质数”。我们也会看到一个更加强大的人工智能。但这并不代表它具备了通用人工智能的能力。

但是，我们需要的不是被定义的 “通用人工智能”，而是一个能够解决实际需求的工具，ChatGPT 对自然语言优秀的处理能力会带来一个契机，一个像互联网一样改变我们生活方式的契机。
